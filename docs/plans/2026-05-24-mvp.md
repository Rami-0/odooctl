# odooctl MVP Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Build a CLI-first open-source Odoo DevOps platform skeleton with reliable config parsing, Docker Compose/PostgreSQL/filestore adapters, staging clone sanitization, module updates, health checks, metadata, examples, docs, and tests.

**Architecture:** Python package using Typer for CLI, Pydantic for YAML validation, Rich for operator output, and subprocess wrappers for scriptable Docker/PostgreSQL/Odoo operations. Commands are thin orchestration layers over testable adapters/services.

**Tech Stack:** Python 3.11+, Typer, Pydantic v2, PyYAML, Rich, pytest.

---

## Execution Slices

1. Repository/bootstrap: package metadata, CLI entrypoint, tests.
2. Config: schema, default init file, env-secret resolution without committing secrets.
3. Shell/runtime adapters: safe subprocess wrapper, Docker Compose, Postgres, filestore.
4. Odoo operations: healthcheck, sanitization, module update.
5. Metadata and commands: init/deploy/backup/restore/clone/update-modules/rollback/logs/status.
6. Examples and docs.
7. Full verification and commit.

## Verification Gates

- Unit tests for config, shell redaction, backup manifest, clone sanitization orchestration, module update failure behavior.
- CLI smoke tests for `--help`, `init --dry-run`, and status with example config.
- Security review: no committed secrets; logs redact sensitive env values.
- Git commits after stable checkpoints.
