# Getting help

- **Documentation** — <https://rami-0.github.io/odooctl/docs/> covers install,
  configuration, backup/restore, staging clones, deploys, rollback,
  migration rehearsal, and the API.
- **Questions & ideas** — use
  [GitHub Discussions](https://github.com/Rami-0/odooctl/discussions).
  Issues are for confirmed bugs and scoped feature requests.
- **Bugs** — open a
  [bug report](https://github.com/Rami-0/odooctl/issues/new?template=bug_report.yml)
  with your `odooctl --version`, Odoo version, execution mode, and the full
  (redacted) command output.
- **Security vulnerabilities** — never open a public issue; report privately
  per [SECURITY.md](SECURITY.md).
- **Commercial licensing** — see [LICENSING.md](LICENSING.md);
  contact hi@ramishalabi.xyz.
